Quarto

Quarto is an open-source tool for storing and writing about our data work.

  • Why are we using Quarto (instead of Word, Google Docs, Overleaf, etc)?
    • You can put all your work (code, output, and write-up) in one document which allows for a fully reproducible workflow.
    • When you render the Quarto file, you get a slick output document. Some of the output options include PDF, HTML, and MS Word.
    • You can insert math symbols (with what is called Latex code): \(\mu\).
  • When editing a Quarto document, you can choose between the Source editor or the Visual editor. We recommend testing out both.
    • Source editor: Shows the raw markdown syntax when editing.
    • Visual editor: Provides a word processor type interface when editing.

Packages

  • Base R contains lots of useful commands to analyze data. However, people have developed new commands which are not packaged in this standard version of the program. Therefore, people create packages which contain these new commands.

  • Whenever we want to use the commands from a package, we need to load the package.

  • Below is the code that loads the leaflet package in your Quarto document. You will need to run the code (as described above) if you want leaflet to be loaded in your RStudio Session.

library(leaflet)

  • Go to the Packages tab in the lower right. Notice that leaflet is listed and if you ran the code above, then it is loaded so the box next to leaflet should be checked.

  • You may want to use a package that is not listed in the Packages Tab. In this case, you will need to install it first, which can be done using the “Install” button (if the package is shared via the Comprehensive R Archival Network).

  • We have pre-installed on the server the packages we will need today.

Forest Inventory and Analysis Program

Our first dataset comes from the Forest Inventory and Analysis (FIA) Program of the US Forest Service. FIA is tasked with monitoring the status and trends of forests in the US. Across the US, they collect data on trees using a systematic sampling design where they sample a plot of land about once every 6,000 acres. Let’s use their estimates of biomass and trees per acre to make maps in Pennsylvania.

Choropleth map of forest attributes in PA

Let’s explore what each line of the code is doing.

Now to add the value of a forest attribute for each county.

The dataset contains the variables biomass, which is the average tons of aboveground biomass per acre, and trees_per_acre, which is the average number of trees per acre. Let’s incorporate biomass into our map.

ggplot(data = pa_county_trees,
       mapping = aes(x = long, y = lat, group = group, fill = biomass)) +
  geom_polygon(color = "olivedrab") + 
  coord_quickmap() +
  theme_void()

How could we improve on the default color palette?

# Try the viridis option
ggplot(data = pa_county_trees,
       mapping = aes(x = long, y = lat, 
                     group = group,
                     fill = biomass)) +
  geom_polygon(color = "olivedrab") + 
  coord_quickmap() +
  theme_void() +
  scale_fill_viridis_c(direction  =  -1)
# Change to a Greens palette
ggplot(data = pa_county_trees,
       mapping = aes(x = long, y = lat, 
                     group = group,
                     fill = biomass)) +
  geom_polygon(color = "grey34") + 
  coord_quickmap() +
  theme_void() +
  scale_fill_distiller(palette = "Greens",
                       direction = 1)

How should we change the code to get a map of trees per acre instead of biomass?

ggplot(data = pa_county_trees,

  geom_polygon(color = "grey34") +

  theme_void() +
  scale_fill_distiller(palette = "Greens",
                       direction = 1)

Create a simple map

Now let’s create a simple map of the locations of the trees.

bucknell_trees %>%
  leaflet(options = leafletOptions(minZoom = 16, maxZoom = 18)) %>%
  addTiles() %>%
  addMarkers(lat = ~y, lng = ~x)

Add better icons

We can improve upon the icons and label the tree species. Let’s play around with some options.

# Add tree icon
treeIcon <- makeIcon(
  iconUrl = "images/tree.png",
  iconWidth = 30, iconHeight = 25
)


# Create the map with the new icon
bucknell_trees %>%
  leaflet(options = leafletOptions(minZoom = 16, maxZoom = 18)) %>%
  addTiles() %>%
  addMarkers(lat = ~y, lng = ~x, label = ~tree_species,
             icon = treeIcon)
# Another tree icon
treeIcon2 <- awesomeIcons(icon = 'tree',
                      library = 'fa',
                      markerColor = 'darkgreen')

# Create the map with the new icon
bucknell_trees %>%
  leaflet(options = leafletOptions(minZoom = 16, maxZoom = 18)) %>%
  addTiles() %>%
  addAwesomeMarkers(lat = ~y, lng = ~x, label = ~tree_species,
             icon = treeIcon2)

Notice that when you hover over one of the icons, you can see what the species of the tree.

Add more information to the label

The Engineering 100 students captured lots of useful information about these trees! Let’s add that information to the tree labels.

# Better label
label_text <- glue(
  "<b> Tree Species: </b> {bucknell_trees$tree_species} <br/>",
  "<b> Condition: </b> {bucknell_trees$tree_condition} <br/>",
  "<b> Sun: </b> {bucknell_trees$sun_exposure} <br/>",
  "<b> Trunk Circumference (cm): </b> {bucknell_trees$trunk_circumference_cm} <br/>"
) %>%
  lapply(htmltools::HTML)


# Recreate the map
bucknell_trees %>%
  leaflet(options = leafletOptions(minZoom = 16, maxZoom = 18)) %>%
  addTiles() %>%
  addAwesomeMarkers(lat = ~y, lng = ~x, label = label_text,
             icon = treeIcon2)
